# Vibe Coding Rule

## 读文档策略
- 优先读当前 milestone 文档 + 相关契约文档，不整篇回读主 plan。
- 在需求冲突、边界不清、或你要求全局复核时再回读主 plan。

## 开工前
- 声明本次使用的文档清单（2-4 个文件），核对实现与目标、边界、验收标准一致。
- 若当前 milestone 缺少目标、验收或边界任一项，先发起最多 5 个澄清问题。

## 卡住处理
- 同一问题连续 2 次失败时停下，复盘已尝试方案，提出新拆解或 MVP fallback。

## 里程碑收尾
- 验收通过后提示你提交，并建议打里程碑标签；提交需你确认。
- 需求变更先更新文档，再改代码。

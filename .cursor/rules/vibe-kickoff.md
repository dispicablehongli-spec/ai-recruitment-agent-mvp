# Vibe Kickoff Rule

帮你在规划阶段把需求、架构、风险想清楚。

## 澄清引导
进入实现前，AI 主动追问，单次提问 ≤ 5 条，分轮进行直到信息够用：
- 目标用户与使用场景
- 核心问题与成功标准
- 非目标 / 边界
- 输入数据与输出形态
- 外部依赖与集成
- 技术与运行环境约束
- 时间 / 资源约束

## 计划与反思
- 产出分阶段计划：每个 milestone 含输入 / 输出 / 验收。
- 冻结前从 4 个维度做一次反思，发现遗漏或冲突就回到澄清：
  - 需求自洽：成功标准可验收，非目标清晰
  - 架构合理：复杂度与 MVP 匹配，关键选型与权衡明确
  - 工程闭环：范围、依赖、可交付物、可测试性完整
  - 风险可控：Top 3 风险、监控信号、降级或回退路径
- 可切换不同 LLM 做交叉检查；结论冲突时整理冲突点请你确认。

## 构建建议
- 首次构建：完整主 plan 一次性给模型，优先保证涌现质量。
- 后续迭代：另起独立 plan，注明与主 plan 的依赖关系；主 plan 保持只读不膨胀。
